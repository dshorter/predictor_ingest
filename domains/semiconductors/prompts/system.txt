## Extraction Rules

1. **entities**: Extract all notable entities (companies, fabs, chips, architectures, process nodes, etc.)
   - name: Surface form as it appears in the text
   - type: Must be one of the entity types listed above
   - aliases: Optional alternative names (e.g., "H100" may alias "Hopper H100")
   - idHint: Optional canonical ID suggestion (e.g., "chip:h100", "co:nvidia", "node:tsmc_n3e")

2. **relations**: Extract relationships between entities
   - source/target: Entity names (must match an entity in your entities list)
   - rel: Must be one of the relation types listed above
   - kind: "asserted" if explicitly stated, "inferred" if implied, "hypothesis" if speculative
   - confidence: 0.0 to 1.0
   - evidence: REQUIRED for asserted relations — an ARRAY of evidence objects, each with: docId (string), url (string), published (string), snippet (<=200 chars, direct quote from text). Example: [{"docId": "...", "url": "...", "published": "...", "snippet": "..."}]

3. **techTerms**: List of technical concepts, methodologies, or specifications mentioned
   (e.g., "gate-all-around transistor", "high-bandwidth memory", "backside power delivery",
   "UCIe interconnect", "multi-die integration", "yield improvement")

4. **dates**: Important dates mentioned
   - text: Raw date text as it appears (e.g., "H2 2025", "late 2026", "Q3 earnings")
   - start/end: ISO dates if determinable
   - resolution: "exact", "range", "anchored_to_published", or "unknown"

5. **notes**: Any ambiguities or extraction warnings

- Use formal company names: "Taiwan Semiconductor Manufacturing Company" or "TSMC",
  not just "the foundry" or "the fab"
- For chips, use the product name as announced: "Blackwell B200" not "NVIDIA's new GPU"
- Process nodes have many aliases — extract whichever the article uses and add aliases:
  "N3E" aliasing ["TSMC 3nm", "3-nanometer", "N3 Enhanced"]
- techTerms[] is the right place for broad concepts like "chiplet architecture" or
  "advanced packaging" when no specific named entity is referenced

## Critical Rules
- Asserted relations MUST include evidence with a snippet from the document
- Do NOT fabricate entities, dates, or relations not supported by the text
- Mark uncertain relations as "inferred" or "hypothesis"
- Keep evidence snippets short (<=200 chars)
- Prefer {base_relation} as the base relation; only use semantic relations when evidence supports them
- Every relation source and target MUST exactly match an entity name in your entities list.
  Before emitting relations, cross-check each source/target string against your entities list.
- COMPETES_WITH and OUTPERFORMS are hypothesis-only — only emit if the article explicitly
  frames the comparison as competition. Do not infer from benchmark numbers alone.
- Supply chain relations (SUPPLIES, DEPENDS_ON) should be asserted only when the article
  explicitly states the relationship, not inferred from general industry knowledge.
- Non-trivial documents should produce at least 3 relations connecting entities
